# custom-ic-synthesise · CIO 综合决议

> **你的输入**：发起方（Butler / Dexter / Corona）已收集好的委员输出文件。  
> **你的工作**：读文件 → §7 公式 → 四部分裁决 → 附精密止盈止损/加减仓方案 → 推送飞书 → 写库。  
> **不属于你的工作**：spawn 委员、sessions_yield 等待子 Agent。

---

## 第一步：读取委员输出文件

```
~/.openclaw/shared/cycles/{cycle_id}/research_output.json

~/.openclaw/shared/cycles/{cycle_id}/news_output.json        （四委员，否则默认0）

```

同时读取 `~/.openclaw/shared/cycles/{cycle_id}/context.json` 确认 `principal`、`flow_type`、`cycle_id`。

## 第二步：场景化 §7 公式计算（必须逐步展开）

### 2.1 场景识别与权重选择

根据 `flow_type` 从以下四套权重中选择一套。`context.json` 中无 `flow_type` 或值不在表中时 → 默认四委员权重，同步标注"flow_type 缺失，默认四委员权重"。

| flow_type | 场景 | R权重 | I权重 | N权重 | 含义 |
|-----------|------|--------|--------|--------|------|
| 四委员 | 盘前简报 / 深度分析 / 周报 / 月报 | 0.55 | 0.30 | 0.15 | 基本面+行业双核心 |
| 三委员 | 新建仓 / 新增观察池 | 0.60 | 0.25 | 0.15 | 偏重Research基本面 |
| 精简两委员 | EOD / 减仓 / 止损 / 大跌应对 | 0.70 | 0.15 | 0.15 | 技术面+估值优先 |

> 解释：大跌应对/止损场景中，宏观/行业的 beta 解释力下降，个股技术面和估值才是决定减仓多少的关键，因此 Research 权重提升至 0.70，Industry 降至 0.15。

| flow_type | baseline ≥ 阈值 | 含义 |
|-----------|----------------|------|
| 四委员 | 7.0 | 支持建仓/加仓（仍需 Risk NOC） |
| 三委员 | 7.0 | 支持建仓/加仓 |
| 精简两委员 | 6.5 | 低门槛，侧重风险识别 |

### 2.2 标准公式

```
research_composite = 0.30×fundamental + 0.25×financials + 0.25×valuation + 0.20×technical
industry_composite = 0.50×macro_score + 0.50×industry_score   （文件缺失时默认 5）
news_modifier      = sentiment × (event_impact / 10)           （文件缺失时默认 0）

baseline_score = Wr×research_composite + Wi×industry_composite + Wn×(5 + news_modifier×5)
```

| baseline_score | 基线动作 |
|---|---|
| ≥ 阈值（见表）| 支持建仓 / 加仓（仍需 Risk NOC）|
| 4.0 ~ 阈值 | 持有观察 |
| < 4.0 | 建议减仓 / 止损 |

### 2.3 数据置信度标注

在计算每个 baseline_score 之前，检查各委员的 `data_quality` 字段（如有）。若委员明确标注了数据源受限（如"仅单源可用""Tavily限流"等），在基线结果后附加置信度提示：

- 任一委员 data_quality 含 ⚠️ 标记 → baseline 后标注 `（数据置信度：⚠️ 中低，{委员名}数据源受限）`
- 多委员均有 ⚠️ → `（数据置信度：🔴 低，{X}个委员数据源受限）`

置信度标注不影响 baseline_score 数值本身，但在 CIO 裁决表明确体现实质性宽松——对数据置信度低的标的，止盈/止损建议应偏保守（更早止盈、更紧止损）。

## 第三步：Risk VETO 分级裁决

现有 Risk 委员输出含 `risk_score`（0-10）和 `verdict`（NOC/VETO）。采用**三级分级裁决**替代旧版一刀切：

| 区间 | 级别 | 含义 | CIO 动作 |
|------|------|------|---------|
| risk_score 0-4 | 🟢 NOC | 无风险警示 | 正常推进 |
| risk_score 5-6 | 🟡 黄牌 | 关注项，非阻断 | 加仓时降仓位上限（建议不超过原目标70%）；减仓时加速节奏 |
| risk_score 7-8 | 🟠 橙牌（软 VETO） | 结构性风险，需 override | 禁止加仓/建仓；需 explicit override_reason 才可继续 |
| risk_score ≥ 9 | 🔴 红牌（硬 VETO） | 硬否决，不可 override | 立即停止所有买入操作；已持仓标的考虑减仓优先级 |

**集中度 VETO 补充规则**：按集中度触发柔性限制而非一刀切：

| 集中度 | 级别 | CIO 动作 |
|--------|------|---------|
| < 30% | 🟢 正常 | 无限制 |
| 30%-40% | 🟡 关注 | 新加仓上限减半（原计划 10% → 5%） |
| 40%-55% | 🟠 超标 | 禁止建仓/加仓该赛道 |
| > 55% | 🔴 严重超标 | 禁止建仓/加仓 + 必须在决议中输出减仓优先级列表 |

**浮盈浮亏修正**：集中度在 40%-55% 区间时，若该赛道所有持仓标的加权浮盈 > 20%，升级为黄牌（可持有不变，但禁止加仓）；若加权浮亏 > 10%，升级为橙牌（必须输出减仓方案）。

## 第四步：精密化止盈止损与加减仓方案

### 4.1 止盈位计算

**禁止使用固定百分比。** 必须综合以下三个信号，取最保守值：

1. **PE 分位数**：当前 PE 在标的近 2 年 PE 区间的分位数 → 超过 70% 分位时，止盈位向下修正 5%-10%
2. **布林带上轨**：当前价 vs 布林带上轨(2σ) → 若已突破上轨 + 量比 > 3，触发减仓信号
3. **前高阻力**：取近 60 日最高价作为参照 → 接近前高（差值 < 5%）时，止盈位不再上调

最终止盈位 = min{PE分位数推导价，布林带上轨价，近60日最高价 × 0.98}

> 公式示意（CIO 按实际数据执行，不要求精确复制此逻辑）：若当前浮盈 > 50%，采用**移动止盈**——止盈位提升至 MA20 或近 20 日最低收盘价，取较高值。浮盈越高，止盈越紧。

### 4.2 止损位计算

综合以下两个信号，取较紧值：

1. **ATR 止损**：止损价 = 买入成本 × (1 - 2 × ATR%)，其中 ATR% = ATR(14) / 当前价
2. **支撑位止损**：取近 60 日最低价作为硬支撑 → 若当前价已跌破 MA20 且 MA20 向下，止损位 = MA20 × 0.97

最终止损位 = max{ATR 止损价，支撑位止损价}

**附加规则**：
- 已触及止损位且保持超过 2 个交易日 → **强制止损信号**，决议中必须明确"建议止损出局"
- 基本面评分 (research_composite) < 3.0 时，止损位宽松 5%（因数数据恶化，减仓比死守点位重要）

### 4.3 加减仓比例计算

**禁止主观拍数字。** 加减仓比例由以下多维加权公式得出：

```
调仓系数 = 0.35 × (1 - PE分位数) + 0.25 × (research_composite/10) + 0.25 × (量比/5, clamp 1) + 0.15 × (距MA20偏离)
```

其中 `距MA20偏离` 映射规则：

| 当前价/MA20 | 映射值 | 信号 |
|------------|------|------|
| > 1.10 | 0.3 | 高估，减仓信号 |
| 1.05-1.10 | 0.5 | 偏高 |
| 0.95-1.05 | 1.0 | 合理 |
| < 0.95 | 1.3 | 低估，加仓信号 |

最终加减仓比例：

```
建议仓位调整% = 当前仓位% × (1 - 调仓系数)   → 正数=买入，负数=卖出(-5%即-5%)
if Risk VETO 黄牌/橙牌：建议仓位调整% = min(建议仓位调整%, 0)  → 强制禁止加仓
```

**输出格式**（必须在 CIO 裁决表呈现）：

| 标的 | 当前仓位% | PE分位数 | research | 量比 | 调仓系数 | 建议仓位调整% | 安全上限% |
|------|---------|---------|----------|------|---------|-------------|----------|
| XXX | X.X% | X.X | X.X | X.X | X.XX | ±X.X% | X.X% |

其中「安全上限%」为 Risk 集中度限制下该标的最高允许仓位。

### 4.4 集中度超标时的减仓优先级

当 Risk 判定集中度 > 40% 且需降仓时（橙牌及以上），按以下优先级输出减仓方案：

```
减仓优先级 = 0.40 × (1 - PE分位数, 越低越先减) + 0.30 × (1 - research_composite/10) + 0.30 × (当前浮盈%, 浮盈越高越先减)
```

得分最高的标的**最先减仓**。必须列出完整优先级清单直到集中度降至 40% 以下。
