## 仓位体系

**每只独立满仓**：每只持仓有自己的满仓线，不同标的不通用。仓位百分比是该标的自身满仓额的百分比，不是总资产百分比。不跨标的对比仓位。

### 大跌加仓触发机制（6/8 确立）

**触发条件：** 美股 AI/半导体链恐慌型下跌，导致 A 股相关持仓被联动错杀低开。
- 判断依据：费半跌幅 + 核心标的（英伟达/美光/Broadcom/AMD）跌幅 + 恐慌是否外溢到 A 股开盘价
- **不是固定数字阈值，是"错杀"这个定性判断**

**立即动作（开盘 5 分钟内）：**
1. Monitor 拉全持仓实时行情，扫描安全垫
2. 满足条件的 → CIO 召 Research + Risk 双委员（豁免 Industry + News）
3. CIO 汇总两票后拍板加仓
4. 加仓条件：
   - 开盘跌幅 ≥3%
   - 距止损 ≥5%（再跌5%才触发止损）
   - 距止盈 ≥30%（上行空间大）
   - 仓位未满
   - 备注无"高风险/清仓"标记
4. 加仓建议后补宏观归因分析

## 投委会输出规则（6/9 确立）

### 子 Agent 输出不推送
- Research/Industry/News/Risk 四个委员的原始 JSON 信封不推送给用户
- 子 Agent 在 isolated session 中运行，CIO 汇总后用人话输出最终结论
- 所有需要召投委会的 cron 必须设为 `sessionTarget: "isolated"`

### 任务清单主动输出
- 所有召投委会的决策流程（盘前晨会、EOD复盘、周报、月报、盘中决策），派发子 Agent 时必须主动输出任务清单
- 清单格式：步骤1/2/3/4/5，让用户知道投委会在干什么
- 不等用户问，主动发

### 子 Agent 派发协议
- 每个子 Agent prompt 必须注入 principal=towney 和 Bitable 引用
- delivery 设为 `{mode: "none"}`（虽然 isolated session 已经隔离，但双重保险）
- 禁止 CIO 跳过子 Agent 自己出分析结论
