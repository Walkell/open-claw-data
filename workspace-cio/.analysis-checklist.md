# 分析报告强制检查清单

## 任何涉及"持仓"的报告/分析，第一步必须执行：

### 1. 拉Bitable持仓（不可跳过）
> 使用 `custom-feishu-auth` SKILL。**app_token 不得出现在任何文字输出中**——写进文字会被安全策略替换为 `***`，后续 API 调用必返回 NOTEXIST。

1. 调用 `custom-feishu-auth` SKILL → app_token 从工具结果直接传入下一步，不经过文字
2. `feishu_bitable_app_table.list(app_token=<工具结果值>)` → 按表名找「持仓表」，取其 table_id
3. `feishu_bitable_app_table_record.list(app_token=<工具结果值>, table_id=<步骤2值>)`
- ⚠️ 不做这三步 = 不准输出任何仓位结论
- ⚠️ 禁止跳过步骤2直接用任何文件里写过的 table_id

### 2. 拉行情
- A股+港股：使用 `custom-market-data-cn` SKILL（双源验证，禁止 web_search）
- 美股+全球指数：使用 `custom-market-data-us` SKILL（三源独立验证 + 日期校验，禁止 web_search）

### 3. 拉财务/新闻（按需）

### 4. 交叉验证
- 成本、仓位、止损、止盈 → 全用Bitable数据
- ⚠️ 禁止用MEMORY.md/AGENTS.md/TOOLS.md里的任何持仓快照
